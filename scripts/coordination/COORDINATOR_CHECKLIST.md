# Coordinator Checklist

Use this before assigning work and before accepting a handoff.

## Before Assignment

- Confirm the task has one owner.
- Confirm the expected output is concrete.
- Confirm the worker has the relevant file paths.
- Confirm the stop condition is explicit.
- Check whether the task depends on secrets, network access, or another worker.

## During Execution

- Keep `TASK_BOARD.md` current.
- Convert vague updates into explicit blocker or progress notes.
- Split work if one task accumulates multiple independent deliverables.
- Escalate quickly if two workers appear to be editing the same files.

## Before Accepting Handoff

- Verify the handoff names changed files.
- Verify the handoff states what was tested.
- Verify blockers are captured if the task is not fully done.
- Link the final artifact, note, or PR back into `TASK_BOARD.md`.
